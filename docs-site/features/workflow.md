# 工作流审批

工作流用于把通用审批能力从业务模块中抽出来。业务模块只需要发起流程、同步状态、查看流程记录，不需要重复实现审批引擎。

## 核心能力

- 流程定义。
- 流程画布。
- 节点配置。
- 条件分支。
- 抄送节点。
- 流程发布和版本。
- 发起审批。
- 我的待办、我的已办、我的申请、我的抄送。
- 同意、驳回、撤回、转办、催办。
- 流程详情和流转日志。
- 抄送已读/未读追踪。

## 流程定义

流程定义描述审批怎么走。

常见节点：

| 节点 | 作用 |
| --- | --- |
| 开始节点 | 流程入口 |
| 审批节点 | 生成待办任务 |
| 条件节点 | 根据表单数据选择分支 |
| 抄送节点 | 通知相关人员知晓 |
| 结束节点 | 流程完成 |

连线表示节点之间的流转方向。条件连线带有规则，默认连线负责兜底。

## 发起审批

发起审批需要：

- 流程编码。
- 业务标识。
- 审批标题。
- 表单 JSON。

示例：

```json
{
  "days": 5,
  "reason": "平台验收测试",
  "type": "测试"
}
```

## 审批处理

审批人可以在 `我的待办` 中处理任务。

处理动作：

- 同意：进入下一节点或结束。
- 驳回：流程结束为驳回状态。
- 转办：把任务交给其他人。

发起人可以在 `我的申请` 中查看流程，必要时撤回或催办。

## 抄送追踪

抄送节点会生成独立抄送记录。

被抄送人可以：

- 在 `我的抄送` 中查看记录。
- 按已读或未读筛选。
- 打开详情后自动标记已读。

管理员或相关人员可以在流程详情中查看抄送回执。

## 接入业务模块

业务模块接入工作流时，不要把审批逻辑写死在业务页面中。推荐方式：

1. 业务模块保存自己的单据。
2. 提交审批时调用工作流服务。
3. 工作流记录业务类型和业务标识。
4. 工作流状态变化后调用业务状态处理器。
5. 业务页面只展示审批状态和流程详情入口。

## 验收建议

上线前至少验证：

- 发起审批。
- 同意。
- 驳回。
- 撤回。
- 转办。
- 催办。
- 抄送已读。
- 消息跳转到流程详情。
