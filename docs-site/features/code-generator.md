# 代码生成器

代码生成器用于降低新增 CRUD 模块的重复劳动。它不是为了替代开发者，而是把稳定、重复、容易出错的样板代码交给模板。

## 能力范围

- 读取数据库表结构。
- 字段选择和字段配置。
- 生成后端实体、DTO、应用服务和接口。
- 生成前端 API、列表页和表单。
- 生成菜单和权限种子。
- 生成历史记录。
- 产物治理和清理。
- 可选审批接入提示。

## 适合生成的模块

适合：

- 标准列表、查询、新增、编辑、删除。
- 字段结构清晰的管理类模块。
- 需要快速接入菜单权限的模块。

不适合直接生成：

- 强交互流程页面。
- 复杂多表事务。
- 大量自定义业务规则。
- 对性能有特殊要求的查询。

这类模块可以先生成基础 CRUD，再手工扩展。

## 生成后检查

每次生成后建议检查：

- 后端项目是否能编译。
- API 是否注册。
- 菜单和权限是否初始化。
- 前端页面是否能打开。
- 查询、新增、编辑、删除是否符合预期。
- 租户字段、审计字段和数据权限是否符合模块定位。

## 二开建议

不要为了适配一个特殊业务把代码生成器模板改得很复杂。更稳的方式是：

1. 模板保持通用。
2. 生成基础模块。
3. 在业务模块中手工补特殊逻辑。
4. 如果同类特殊逻辑出现三次以上，再抽象进模板。
