# 文件存储

文件存储用于支撑头像、附件、导入导出文件和业务文件。MiniAdmin 将文件服务抽象出来，避免业务代码关心文件最终存在哪里。

## 核心能力

- 文件上传。
- 文件下载。
- 文件元数据管理。
- 本地存储。
- MinIO 存储。
- 文件状态检查。
- 异常文件处理。

## 存储抽象

业务层依赖文件应用服务或存储接口，不直接写本地路径或对象存储 SDK。

好处：

- 开发环境可以使用本地存储。
- 生产环境可以切换 MinIO。
- 后续接入其他对象存储时，不需要改业务模块。

## 文件元数据

文件记录通常包含：

- 文件名。
- 内容类型。
- 文件大小。
- 存储提供方。
- 存储路径或对象 Key。
- 上传人。
- 上传时间。
- 文件状态。

## 二开建议

新增业务附件时：

1. 业务表只保存文件 ID 或附件关联记录。
2. 不直接保存本地绝对路径。
3. 下载前检查权限。
4. 删除业务记录时明确是否删除文件。
5. 大文件上传需要单独设计大小限制和超时策略。
