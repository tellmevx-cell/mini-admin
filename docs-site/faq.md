# FAQ

## MiniAdmin 是业务系统还是后台底座？

MiniAdmin 是后台底座。它提供认证、权限、租户、工作流、消息、审计、监控、文件和代码生成等平台能力。具体业务模块应该独立接入，不要污染平台层。

## 为什么不用 Vben 自带 docs 包？

Vben 自带 docs 包更适合维护上游框架文档。MiniAdmin 需要自己的官网、使用说明和二开指南，所以单独放在根目录 `docs-site`，避免后续升级 Vben 时混淆。

## 默认账号是什么？

默认开发账号通常是：

```text
admin / 123456
```

如果生产环境使用，请第一时间修改默认密码。

## 本地可以不用 MySQL 吗？

可以。快速体验可以使用 InMemory。需要验证迁移、索引、租户隔离和真实 SQL 行为时，建议使用 MySQL。

## 看不到菜单怎么办？

按顺序检查：

1. 当前用户是否有角色。
2. 角色是否有菜单。
3. 租户套餐是否允许该菜单。
4. 是否需要退出重新登录。
5. 初始化种子数据是否执行。

## 新增业务模块应该从哪里开始？

先看 [新增模块流程](./developer/add-module.md)。如果是标准 CRUD，可以先使用代码生成器，再按项目约定补权限、租户、审计、测试和文档。

## 工作流必须绑定业务模块吗？

不必须。你可以直接在审批中心发起通用流程。业务模块需要审批状态联动时，再通过业务类型和业务标识接入工作流。

## 消息中心只支持站内信吗？

站内信是基础通道。系统也有邮件和 Webhook 投递记录能力，但需要配置外部通道后才会实际发送。

## 文档站怎么发布？

执行：

```powershell
pnpm docs:build
```

构建产物在：

```text
docs-site/.vitepress/dist
```

把这个目录部署到静态站点服务即可。
