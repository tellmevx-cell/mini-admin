## 使用入口

| 入口 | 用途 |
| --- | --- |
| `工作流 / 审批中心` | 发起审批、处理待办、查看申请和抄送 |
| 顶部铃铛 | 快速查看未读消息 |
| `系统管理 / 消息通知中心` | 查看消息、策略、模板、订阅和投递记录 |

## 审批中心页签

- 我的待办：审批人处理任务。
- 我的申请：发起人查看自己提交的流程。
- 我的抄送：被抄送人查看知会记录和已读状态。
- 发起审批：选择流程并提交审批。
- 流程实例：管理员查看运行记录。
- 流程定义：配置和发布流程。
- 业务绑定：维护通用业务类型和流程定义关系。
- 我的已办：审批人查看处理历史。

## 基础操作路径

1. 管理员确认流程已发布并启用。
2. 发起人在 `发起审批` 提交流程。
3. 审批人在 `我的待办` 同意、驳回或转办。
4. 发起人在 `我的申请` 查看进度、撤回或催办。
5. 被抄送人在 `我的抄送` 打开详情完成已读。
6. 所有人通过消息中心查看通知和跳转详情。

## 消息中心操作

消息中心包含：

- 我的消息。
- 通知策略。
- 订阅偏好。
- 模板配置。
- 投递记录。

站内信是默认通道。邮件和 Webhook 需要完成外部配置后再启用。

## 常见排查

看不到菜单：

- 检查角色菜单权限。
- 退出重新登录。
- 检查租户套餐限制。

发起后没有待办：

- 检查流程是否启用。
- 检查审批节点是否能解析出审批人。
- 检查流程连线是否完整。

消息没有生成：

- 检查通知策略。
- 检查订阅偏好。
- 检查工作流动作是否实际发生。

抄送没有已读：

- 确认当前用户是被抄送人。
- 从 `我的抄送` 或携带抄送参数的消息入口打开详情。
