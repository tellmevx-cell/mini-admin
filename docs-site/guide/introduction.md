# 项目介绍

MiniAdmin 是一套面向学习、交付和二次开发的企业后台管理系统。它以后端 .NET 分层架构和前端 Vben Admin 为基础，把常见后台底座能力做成可运行、可扩展、可验收的工程。

## 适合谁使用

| 角色 | 可以获得什么 |
| --- | --- |
| 学习者 | 通过真实项目理解认证、权限、租户、工作流、消息、审计和前后端联调 |
| 后端开发 | 参考分层架构、应用服务、仓储、EF Core、测试和接口组织方式 |
| 前端开发 | 参考 Vben 页面、动态菜单、权限按钮、表格表单和工作流画布 |
| 项目负责人 | 直接拿到企业后台基础能力，减少从零搭建平台的时间 |
| 二开团队 | 按固定约定新增模块、扩展流程、接入业务和准备上线验收 |

## 技术栈

| 层级 | 技术 |
| --- | --- |
| 后端 | .NET 10、ASP.NET Core Minimal API、JWT、EF Core、xUnit |
| 数据库 | InMemory、MySQL、EF Core Migrations、初始化种子数据 |
| 缓存 | Memory Cache、Redis 快速降级 |
| 前端 | Vue 3、Vben Admin、Ant Design Vue、Pinia、Vue Router、Vue Flow |
| 文档 | Markdown、VitePress、运行手册、功能文档规范 |

## 项目能力

MiniAdmin 当前覆盖的平台能力：

- 登录认证、JWT、密码策略、在线用户、多会话管理。
- 用户、角色、菜单、部门、岗位、字典、参数。
- RBAC 权限、菜单权限、按钮权限、数据权限和权限诊断。
- SaaS 租户、租户套餐、租户初始化、租户状态控制。
- 文件上传、文件存储抽象、本地存储、MinIO 扩展点。
- 审计日志、实体变更审计、登录日志、操作追踪。
- 系统监控、告警中心、告警规则、计划任务和运行日志。
- 工作流定义、流程发布、发起审批、待办已办、转办、撤回、催办、抄送已读。
- 消息中心、站内信、通知模板、通知策略、订阅偏好、投递记录和失败重试。
- 代码生成器、生成历史、产物治理、通用导入导出能力。

## 项目边界

MiniAdmin 是后台平台底座，不绑定某个业务领域。你可以在它之上接入合同、采购、报销、客户、库存等业务模块，但这些业务模块应该通过独立功能目录、独立菜单和独立权限接入，不要污染平台层。
