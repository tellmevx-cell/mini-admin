# 权限诊断链路增强设计

## 目标

在现有 `系统监控 / 权限诊断` 上增强权限链路解释能力，让管理员能快速判断用户无菜单、404、按钮不显示是角色授权、租户套餐、用户状态还是缓存导致。

## 设计

后端扩展 `PermissionDiagnosticsDto`，新增租户套餐摘要、角色菜单贡献和诊断提示。仓储仍然由 `EfPermissionDiagnosticsRepository` 负责聚合数据：先读取用户与角色，再读取角色菜单集合，再读取当前租户套餐菜单集合，最终计算角色菜单与套餐菜单的交集。

前端保留现有入口和页面基调，增加“诊断结论”“租户套餐上限”“角色贡献”三个区域。页面不做自动修复，只给出明确原因和下一步建议，例如去角色管理分配权限、去租户套餐检查菜单、或刷新缓存。

## 数据流

```mermaid
flowchart LR
    A["输入用户名"] --> B["读取用户与角色"]
    B --> C["统计角色菜单"]
    B --> D["读取租户套餐"]
    C --> E["计算最终菜单"]
    D --> E
    E --> F["生成诊断提示"]
    F --> G["权限诊断页面"]
```

## 测试

新增后端测试覆盖租户用户诊断：返回租户套餐摘要、角色菜单贡献、最终菜单数量和诊断提示。保留现有刷新缓存测试。
