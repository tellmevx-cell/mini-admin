# 工作流审批人范围修复需求

## 背景

流程设计器支持在审批节点上选择用户或角色作为审批人。当前系统已经支持平台和租户两类数据范围，因此工作流审批人也必须遵守相同范围。

用户在平台 Admin 账号下编辑流程时，希望把审批人改成 Admin；同时如果节点误选了租户用户“主管(auditor)”，流程执行时不能只提示“没有可用审批人”，需要明确说明原因和处理方式。

## 目标

- 流程定义页面只展示当前流程范围内可用的审批用户和审批角色。
- 平台流程只能选择平台用户、平台角色。
- 租户流程只能选择当前租户用户、当前租户角色。
- 角色选项必须至少包含一个当前范围内的启用用户。
- 流程执行时，如果无法解析审批人，需要提示具体原因。
- 提示中需要说明当前登录用户不能代替节点审批人审批。

## 数据流

```mermaid
flowchart LR
  A["流程定义页面"] --> B["读取工作流审批人接口"]
  B --> C["按平台/租户过滤用户和角色"]
  C --> D["保存流程定义"]
  D --> E["流程发起或流转"]
  E --> F["按同一范围解析审批人"]
  F --> G["生成待办或返回明确错误"]
```
