# 工作流审批人范围修复总结

## 本次完成

- 修复了流程设计器审批人下拉范围过宽的问题。
- 平台流程现在只会展示平台可用审批用户和角色。
- 租户流程会展示当前租户可用审批用户和角色。
- 后端执行流程时，如果节点审批人不可用，会返回更具体的原因。
- 错误信息会提示不能用当前登录用户代替节点审批人审批，并引导把节点审批人改成 Admin 后重新发起流程。

## 关键判断

工作流审批人不能直接复用系统用户列表。系统用户列表服务于管理视角，可能会展示跨租户数据；工作流执行服务于业务流转，必须和流程实例所在平台/租户范围一致。
