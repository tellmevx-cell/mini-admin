# 代码生成器字段选择总结文档

## 本次完成

本次把代码生成器字段配置改成了“两段式”第一版：

1. 选择数据库表后，先展示该表字段列表。
2. 用户勾选需要生成的字段。
3. 只有已勾选字段进入下方字段配置区。
4. 预览和生成只提交字段配置区中的字段。

## 交互变化

- 新增“表字段选择”区域：
  - 显示列名、数据库类型、.NET 类型、TS 类型、主键/可空/系统字段标识和字段说明。
  - 每行可勾选是否参与生成。
  - 支持“全选业务字段”和“取消全选”。
- 新增“字段配置”区域：
  - 只展示已勾选字段和自定义字段。
  - 可继续调整属性名、显示名、类型、控件和生成位置。
- “新增字段”调整为“新增自定义字段”，语义更清晰。
- 修复数据表行点击选择：
  - Ant Design Vue 表格行点击改为项目已有的 `custom-row` 写法。
  - 顶部“表名”改为可搜索下拉，选择表名后同样会读取表字段。

## 默认规则

选择表后默认勾选普通业务字段。

以下字段默认不勾选：

- 主键字段。
- `id`
- `tenant_id`
- `created_at`
- `updated_at`
- `deleted_at`
- `create_time`
- `update_time`
- `create_by`
- `update_by`
- `is_deleted`

## 页面体验优化

根据 `critique` 检查，当前页面没有明显 AI 生成式视觉反模式，自动检测器返回空结果。主要问题是三列并排导致字段配置和预览区拥挤，主操作分散在不同面板。

本次已优化：

- 顶部增加“代码生成配置”标题和说明，让页面目标更明确。
- 将“预览”和“生成”主操作集中到顶部右侧。
- 权限码移动到配置区下方，减少预览区干扰。
- 左侧数据表和右侧字段生成保持并排。
- 生成预览改为下方通栏，提升文件路径和代码内容的可读性。
- 选中数据表增加当前表摘要，降低用户确认成本。
