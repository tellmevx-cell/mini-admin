# 代码生成器字段选择需求文档

## 背景

代码生成器当前已经可以读取 MySQL 表和字段，并在选择表后自动生成字段配置。但现有交互会把表字段一次性全部放入生成配置区，用户如果只想生成部分字段，需要逐项调整或删除，使用体验不够像企业级代码生成器。

本功能要把字段配置调整为“两段式”：

1. 先从数据库表字段中选择需要生成的字段。
2. 再对已选择字段配置显示名、属性名、控件类型和生成位置。

## 目标

- 选择数据库表后展示该表全部字段。
- 每个字段支持勾选是否参与代码生成。
- 默认采用 A 策略：除系统字段外默认勾选。
- 用户取消勾选字段后，预览和生成不再包含该字段。
- 已勾选字段进入下方字段配置区，可继续调整生成规则。
- 保留自定义字段能力，用于后续扩展或特殊场景。

## 默认勾选规则

默认勾选除系统字段以外的字段。

系统字段默认不勾选：

- `id`
- `tenant_id`
- `created_at`
- `updated_at`
- `deleted_at`
- `create_time`
- `update_time`
- `create_by`
- `update_by`
- `is_deleted`

主键字段默认不勾选。

## 页面交互

### 表字段选择区

- 选择表后展示字段列表。
- 字段列表包含：
  - 勾选框。
  - 列名。
  - 数据库类型。
  - .NET 类型。
  - TypeScript 类型。
  - 是否主键。
  - 是否可空。
  - 字段注释。
- 支持全选/取消全选当前可生成字段。

### 字段配置区

- 只展示已勾选字段和自定义字段。
- 可以继续编辑：
  - 属性名。
  - 显示名。
  - 控件类型。
  - 是否查询。
  - 是否列表显示。
  - 是否新增。
  - 是否编辑。
- 取消上方字段勾选后，下方配置区同步移除该字段。
- 重新勾选字段时，应尽量恢复字段默认配置。

## 数据流

```mermaid
flowchart LR
    A["选择数据库表"] --> B["读取表字段"]
    B --> C["按默认规则勾选字段"]
    C --> D["字段选择区"]
    D --> E["已选字段配置区"]
    E --> F["构建预览请求"]
    F --> G["后端模板渲染"]
    G --> H["预览文件"]
```

## 验收标准

- [ ] 选择表后能看到表字段选择区。
- [ ] 普通业务字段默认勾选。
- [ ] 主键和系统字段默认不勾选。
- [ ] 取消字段勾选后，该字段不进入预览请求。
- [ ] 已勾选字段可以继续配置生成位置和控件类型。
- [ ] 自定义字段仍可新增和删除。
- [ ] 前端构建通过。

## 不做范围

- 不做数据库表字段在线修改。
- 不做字段拖拽排序。
- 不做主子表字段联动。
- 不做自动识别字典字段。
