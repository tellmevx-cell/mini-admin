# 全后台视觉一致性整理需求

## 背景

前一轮 UI 审查只处理了少量明显问题。为了让 MiniAdmin 更像一个完整的企业级后台产品，需要对整个后台页面做统一的视觉整理。

## 目标

- 统一表格页、配置页、监控页和运行管理页的基础视觉语言。
- 保持安静、清晰、工作台式的企业后台风格。
- 不改变业务功能、接口、权限和数据流。
- 优先通过全局样式层解决共性问题，减少逐页复制样式。

## 覆盖范围

- 系统管理常规页面：用户、角色、菜单、部门、岗位、字典、参数、通知、文件、在线用户等。
- 平台管理页面：租户相关页面。
- 业务页面：代码生成器生成出的表格型页面。
- 特殊工作台页面：系统监控、项目运行管理、代码生成器。

## 重点统一项

- 面板边框、圆角、背景和间距。
- 表格头部、行 hover、分页间距和文本换行。
- 工具栏标题、说明文字和操作区布局。
- Modal、Drawer 的标题、表单标签和输入控件细节。
- 移动端工具栏折行和表格密度。
