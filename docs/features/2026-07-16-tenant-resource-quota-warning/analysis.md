# 租户资源用量看板与配额预警分析

## 背景

现有租户资源配额已经覆盖用户创建、批量导入、文件上传和删除释放，但租户管理员只能在写入失败后感知配额不足，平台管理员也缺少统一的风险视图。需要在不改变现有配额校验语义的前提下，补齐用量展示、状态追踪和主动预警。

## 用户与场景

### 租户管理员

- 登录工作台即可看到用户数、存储空间的已用量、上限、百分比和状态。
- 用量达到预警或耗尽阈值时，在消息中心收到站内通知，并实时更新未读数量。
- 可从看板快捷进入用户管理或文件管理释放资源。

### 平台管理员

- 在租户列表快速识别正常、预警、耗尽和不限额租户。
- 查看各资源最近一次预警通知时间。
- 在定时任务页面手动执行扫描并查看每个异常租户的执行明细。

## 业务规则

1. 套餐上限为 `0` 时表示不限额，状态为 `Unlimited`。
2. 使用率低于 `80%` 时状态为 `Normal`。
3. 使用率达到 `80%` 且低于 `100%` 时状态为 `Warning`。
4. 使用率达到或超过 `100%` 时状态为 `Exhausted`。
5. 第一次进入 `Warning` 或 `Exhausted` 时通知租户内启用的 `tenant-admin` 用户。
6. 同一状态连续扫描不重复通知；恢复正常后再次升高可以重新通知。
7. 无接收人时保留待通知状态，后续扫描可重试。
8. 用量按数据库实时统计：用户包含启用和停用账号，存储包含该租户现存的全部文件记录。
9. 仅扫描启用且未过期的租户，平台租户不受套餐配额限制。

## 数据与一致性

- 新增独立预警状态表，不修改租户、套餐、用户和文件主表。
- 每个租户、每种资源只有一条状态记录，通过唯一索引保证幂等。
- 通知使用确定性的来源标识，结合消息表唯一索引避免并发重复。
- 数据库采用正式 EF Core 增量迁移，同时保留启动时兼容建表逻辑和 InMemory 测试支持。

## 范围

### 本期包含

- 租户工作台资源用量看板。
- 平台租户列表状态增强。
- 配额预警状态持久化。
- 每日定时扫描、手动执行和执行明细。
- 消息中心通知与 SignalR 实时推送。
- 默认 Scriban 消息模板。

### 本期不包含

- 套餐购买、续费、自动升级或计费。
- 邮件、短信、Webhook 外部投递。
- 自定义阈值和多级阈值策略。
- 历史用量曲线与容量预测。

## 验收标准

- 租户工作台可正确显示用户和存储实时用量。
- 80% 与 100% 状态边界准确，不限额资源不会告警。
- 状态首次进入风险区时产生站内通知，重复扫描不重复发送。
- 恢复正常后再次进入风险区会再次发送通知。
- 平台租户列表和定时任务明细能够定位异常租户与资源。
- MySQL 迁移、InMemory 测试、后端全量测试和前端构建均通过。
