# Code Generator Existing Module Requirements

## 背景

代码生成器已经支持从 MySQL 业务表生成后端 CRUD、前端页面、菜单权限和生成历史。实际使用时，用户可能选择已经被当前系统实体映射的表，例如 `mini_customer`。此前系统只在预览或生成时返回“不能重复生成模块”的错误，用户不容易提前判断原因。

## 目标

- 表列表和表详情需要识别当前表是否已被系统实体映射。
- 已映射表需要展示对应实体名、模块类型、可跳转路由和关联文件。
- 选择已映射表时，前端应明确提示“维护已有功能或换表生成”，并禁用预览/生成。
- 后端仍需保留预览/生成拦截，避免绕过前端造成重复覆盖。

## 非目标

- 本阶段不做已有模块的自动二次同步。
- 本阶段不做代码文件在线编辑。
- 本阶段不允许覆盖系统内置模块。
