# Redis 快速兜底修复需求文档

## 背景

系统支持 Redis 和内存两种缓存方式。当前开发环境配置了线上 Redis，当 Redis 网络不稳定或响应慢时，后端缓存操作会等待 Redis 超时后才回退到内存，导致登录、验证码、菜单加载等前端关键链路长时间转圈。

## 目标

- Redis 可用时继续优先使用 Redis。
- Redis 不可用或超时时快速回退到内存缓存。
- Redis 第一次失败后，短时间内不重复打 Redis，避免每个接口都等待超时。
- 不影响未配置 Redis 时的内存缓存模式。

## 非目标

- 不在本次实现 Redis 可视化配置管理。
- 不改变线上 Redis 连接字符串的配置入口。
- 不改变登录、菜单、验证码的业务逻辑。

## 验收标准

- 后端启动正常。
- Redis 异常时缓存读写仍可通过内存兜底。
- Redis 异常后后续缓存访问不会反复调用失败的 Redis。
- 后端完整测试通过。
- 前端和后端服务均可访问。
