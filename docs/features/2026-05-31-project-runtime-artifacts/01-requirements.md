# 项目运行管理构建历史与产物管理需求

## 背景

项目运行管理已经支持服务启动、停止、实时日志和服务级打包。为了后续工作流编排，需要沉淀每次打包的执行记录，并让用户在页面上明确看到构建产物在哪里、是否存在、大小和最后更新时间。

## 目标

- 每次服务级打包都生成一条构建历史。
- 构建历史记录开始时间、结束时间、耗时、状态、退出码、日志路径、产物路径和触发命令。
- 构建成功或失败后，页面能看到最近构建结果。
- 页面能展示构建产物是否存在、文件/目录大小、最后更新时间。
- 支持打开产物路径，方便本地查看发布文件。

## 第一版范围

- 历史记录存储在 `data/project-runtime/build-history.json`。
- 产物路径通过服务配置 `buildArtifactPath` 描述。
- 默认 MiniAdmin API 产物路径：`artifacts/publish/mini-admin-api`。
- 默认 Vben Web 产物路径：`frontend/vue-vben-admin/apps/web-antd/dist.zip`。
- 前端在“项目运行管理”页面展示当前服务的最近构建和产物信息。

## 非目标

- 第一版不上传产物到远程服务器。
- 第一版不做构建记录数据库化。
- 第一版不做下载大文件接口，只打开本机路径或展示路径。
- 第一版不做复杂构建流水线编排。
