# 消息模板中心需求

## 背景

消息中心已经具备站内信、投递记录和通道概览能力，但通知标题、内容和跳转链接仍分散在业务代码中硬编码。后续工作流、告警、业务事件都会持续增加通知类型，需要一个统一的模板配置入口。

## 本阶段范围

1. 在通知中心增加“模板配置”页签，支持查询、预览、编辑、启停模板。
2. 后端新增消息模板实体、仓储、渲染服务和 API。
3. 默认种子消息模板覆盖系统告警和工作流常用通知。
4. 系统告警站内信优先接入模板渲染。
5. 工作流通知模板完成默认数据、页面配置和执行链路迁移。

## 非本阶段范围

1. 不在本阶段接入短信、企业微信、飞书等外部通道。
2. 不在本阶段为邮件和 Webhook 单独设计一套模板格式；外部通道先复用同一套文本模板和投递记录。
3. 不在本阶段做模板变量强校验；未知变量继续原样保留，避免误拦截已有通知。
4. 不在本阶段做富文本编辑器，模板先使用文本和占位符。

## 模板变量

模板通过 `{变量名}` 引用运行时变量。当前默认示例变量包括：

- `title`
- `content`
- `levelText`
- `instanceTitle`
- `definitionName`
- `nodeName`
- `operatorUserName`
- `comment`

未知变量会原样保留，便于配置时发现缺失变量。

## 收口判断

当前阶段以“站内信模板可配置、工作流通知可渲染、模板缺失可兜底”为完成标准。邮件/Webhook 独立模板、变量强校验和富文本编辑都属于增强项，当前不是业务上线前的必须功能。
