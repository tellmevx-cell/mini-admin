# 字典管理需求文档

> 回补整理。

## 背景

后台系统中有大量枚举类数据，例如用户状态、通知类型、是否启用。字典管理用于把这些可配置项从代码中抽出来，方便后续扩展和维护。

## 目标

- 字典页面采用左侧字典类型、右侧字典项布局。
- 点击左侧字典类型后，右侧展示对应字典项。
- 支持字典类型 CRUD。
- 支持字典项 CRUD。
- 支持启用状态、排序、标签和值。

## 功能范围

- 字典类型列表。
- 字典类型新增、编辑、删除。
- 字典项新增、编辑、删除。
- 左右联动布局。

## 交互流

```mermaid
flowchart LR
    A["左侧字典类型"] --> B["点击用户状态"]
    B --> C["右侧加载该类型字典项"]
    C --> D["新增/编辑/删除字典项"]
```

## 验收标准

- [x] 字典页面是左右布局。
- [x] 点击字典类型后右侧数据会变化。
- [x] 能给用户状态等字典增加子项。
- [x] 字典类型有子项时删除受保护。
