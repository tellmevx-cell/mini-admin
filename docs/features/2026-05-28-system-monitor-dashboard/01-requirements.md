# 系统监控看板需求文档

## 背景

当前系统已经具备 MySQL、Redis/内存缓存、Local/MinIO 文件存储、定时任务、操作日志、登录日志和在线用户等能力，但缺少一个统一入口让管理员快速判断系统运行是否健康。

系统监控看板用于把运行时、服务器、关键依赖和最近异常集中展示，帮助管理员完成基础排查。

## 目标

- 在 `系统监控` 菜单下新增 `系统监控` 页面。
- 页面风格符合当前 MiniAdmin/Vben：浅灰背景、白色信息块、状态 Tag、克制按钮、高密度信息展示。
- 后端提供统一监控摘要 API。
- 展示 API、MySQL、Redis/内存缓存、文件存储的健康状态。
- 展示 CPU、内存、GC、线程、运行时长等运行指标。
- 展示服务器和应用信息。
- 展示最近状态：任务失败数、异常操作日志数、在线用户数、异常文件数。

## 页面结构

- 顶部摘要：API 状态、MySQL、缓存、文件异常。
- 中部双列：CPU、内存。
- 中部整块：应用与服务器信息。
- 底部双列：依赖健康检查、最近状态。

## 权限

- 新增菜单权限：`system:monitor:query`。
- Admin 默认拥有该权限。

## 不做范围

- 不做历史趋势图。
- 不做 Prometheus/Grafana 集成。
- 不做服务器级真实 CPU 总占用率采样守护进程。
- 不做自动告警推送。

## 验收标准

- [x] 管理员可以打开 `系统监控 > 系统监控` 页面。
- [x] 后端 `GET /system/monitor/overview` 返回监控摘要。
- [x] API 状态、MySQL、缓存、文件存储健康状态可展示。
- [x] CPU、内存、GC、线程、运行时长可展示。
- [x] 服务器和应用信息可展示。
- [x] 最近任务失败数、异常操作日志数、在线用户数、异常文件数可展示。
- [x] 后端完整测试通过。
- [x] 前端构建通过。
