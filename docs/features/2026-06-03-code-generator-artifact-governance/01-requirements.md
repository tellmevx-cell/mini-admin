# Code Generator Artifact Governance Requirements

## 背景

代码生成器在开发环境中写入前端文件时，Vite 可能触发刷新。如果生成过程被中断，可能出现“文件已生成、表已映射、但没有生成历史”的半生成状态。误选系统表也会生成不适合维护的业务模块。

## 目标

- 提供产物治理页签，扫描已有生成文件。
- 展示模块是否有生成历史、菜单权限、实体映射、风险原因和关联文件数量。
- 支持对无历史产物补登记生成历史。
- 支持清理无历史产物的生成文件和菜单权限。
- 对 `__EFMigrationsHistory`、代码生成历史表、种子版本表等系统维护表阻止生成。

## 非目标

- 不自动删除业务表，清理默认保留数据表和业务数据。
- 不自动判断业务字段语义。
- 不替代已有生成历史的正式回滚流程。
