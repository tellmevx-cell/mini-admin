# Code Generator Artifact Governance Summary

## 完成内容

- 代码生成器现在会阻止系统维护表生成业务模块，例如 EF 迁移历史表。
- 新增产物治理能力，可以看到由生成器产出的模块是否缺少生成历史。
- 对没有历史的产物，可以选择补登记历史，后续就能在生成记录中追踪。
- 对没有历史且不需要保留的产物，可以清理生成文件和菜单权限。

## 使用建议

如果看到类似 `Efmigrationshistory` 这种系统表误生成模块，建议在“产物治理”页签中清理。清理默认不会删除业务表，避免误删数据。

## 后续方向

下一步可以增强为“清理前预览删除清单”和“补登记时选择是否启用导入导出/审批接入”。
