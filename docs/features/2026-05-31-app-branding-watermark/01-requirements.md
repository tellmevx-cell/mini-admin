# 应用品牌与全局水印需求

## 背景

当前前端默认标题仍来自 Vben 模板配置，显示为 `Vben Admin Antd`。企业级后台需要支持运行期配置品牌名称，并支持管理员开启全局水印，用于在后台主界面形成操作留痕提示和截图传播提醒。

## 目标

- 后台品牌名称不再只能依赖前端 `.env`，需要可由管理员在系统参数中配置。
- 登录页、浏览器标题、后台布局内展示的应用名称应尽量使用同一份后端配置。
- 管理员可以通过系统参数控制是否开启全局水印。
- 管理员可以通过系统参数自定义水印文字。
- 未配置或参数被禁用时，系统使用安全默认值，保证登录和主界面可用。

## 范围

- 使用现有 `mini_system_parameters` 存储配置，不新增数据表。
- 新增公开接口 `GET /public/app-branding`，登录前也可以读取。
- 前端启动时读取品牌配置，并应用到 Vben preferences。
- 水印只在登录后的后台主布局生效，不覆盖登录页。

## 参数约定

| 参数键 | 默认值 | 说明 |
| --- | --- | --- |
| `app.brand.name` | `MiniAdmin` | 系统完整名称 |
| `app.brand.shortName` | `MiniAdmin` | 短名称，用于登录页等空间有限位置 |
| `app.brand.loginTitle` | `MiniAdmin 企业后台` | 登录页标题 |
| `app.brand.copyright` | 空 | 版权或页脚文案，预留 |
| `app.watermark.enabled` | `false` | 是否启用全局水印 |
| `app.watermark.text` | 空 | 自定义水印文字 |

## 非目标

- 水印不是安全防护能力，不能阻止截图或复制。
- 本阶段不新增专门的品牌配置页面，先复用参数设置 CRUD。
