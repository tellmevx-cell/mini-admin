# 用户管理与部门树筛选完工总结

## 完成内容

- 用户管理页面改为左侧部门树、右侧用户表格。
- 用户列表支持按部门和岗位过滤。
- 用户新增/编辑支持部门、岗位和多角色。
- 用户表格展示部门、岗位、角色、状态等信息。
- 增加自删除保护，避免登录用户删除自己。
- 与数据权限联动，限制跨部门编辑和删除。

## 关键实现

- `EfUserRepository` 聚合用户、部门、岗位、角色信息。
- 前端点击部门树节点后将 `departmentId` 带入用户列表请求。
- 用户保存时用角色 id 列表重建 `UserRole` 关系。
- 删除用户前由应用服务判断当前用户和目标用户。

## 使用方式

进入 `系统管理 / 用户管理`，左侧选择部门，右侧表格会展示该部门下的用户。新增或编辑用户时，可以选择部门、岗位和角色。
