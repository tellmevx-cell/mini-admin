# 租户到期时间与登录选项防呆需求

## 背景

创建租户并执行初始化模板后，用户退出登录，在登录页没有看到新创建的机构选项。

排查发现租户本身创建成功，初始化状态也是 `Success`，但 `ExpireAt` 被设置为当天凌晨，实际已经早于当前时间。登录页租户选项只展示启用且未过期的租户，因此该租户被过滤。

## 目标

- 管理端租户列表应展示真实有效状态：启用租户如果到期时间已过，应显示为 `已过期`。
- 新增租户时，如果填写到期时间，必须晚于当前时间。
- 编辑租户时，如果填写到期时间，必须晚于当前时间。
- 前端表单应尽量避免选择过去日期，并在提交前给出明确提示。
- 登录页仍只展示启用且未过期的租户，保证过期租户无法被继续选择登录。

## 非目标

- 本次不做租户自动续期。
- 本次不做租户过期后的自动任务清理。
- 本次不改变租户套餐和菜单权限逻辑。
