# 租户登录选项和默认菜单修复需求文档

## 背景

新增 `jxnc` 租户后，登录页下拉仍然只显示 `Admin` 和 `demo`。用户手动填写租户编码后，租户管理员登录可能出现没有菜单、无法进入系统的情况。

## 目标

- 登录页下拉从后端读取当前可登录租户，不再只写死 `Admin/demo`。
- 新增租户后，租户管理员角色必须具备默认工作台菜单。
- 系统启动时清理租户管理员权限缓存，避免旧的空菜单缓存影响登录。
- 保留平台管理员 Admin 的快速登录入口。

## 验收标准

- `/auth/tenant-options` 可匿名返回启用且未过期的租户。
- 登录页下拉包含 `demo`、`jxnc` 等活跃租户。
- 新增租户管理员登录后至少能看到工作台入口。
- 后端相关测试、完整测试、前端构建通过。
