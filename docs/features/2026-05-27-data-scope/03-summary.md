# 数据权限完工总结

## 完成内容

- 角色支持配置数据范围。
- 用户列表会按当前登录用户数据范围过滤。
- 用户编辑、删除会校验目标用户是否在授权范围内。
- 跨部门删除会返回“没有权限删除其他部门账户”。
- 权限诊断中心可以展示用户最终数据范围。

## 关键实现

- `Role.DataScope` 保存角色数据权限。
- `EfDataScopeProvider` 负责把用户角色解析成最终数据范围。
- 用户管理 Repository 负责查询侧过滤。
- 用户应用服务负责命令侧保护，避免绕过列表直接调用接口。

## 设计取舍

当前先实现 `All` 和 `Department`，因为它们覆盖了学习阶段最常见的企业场景。`DepartmentAndChildren`、`Self`、`Custom` 可以在部门树稳定后继续扩展。

## 后续建议

- 增加“本部门及以下”。
- 增加自定义数据权限部门选择。
- 在更多业务模块复用 `IDataScopeProvider`，形成统一数据权限策略。
