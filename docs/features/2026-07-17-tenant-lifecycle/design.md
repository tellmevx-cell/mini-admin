# 租户生命周期闭环设计

> 关联设计：[租户资源用量与配额预警](../2026-07-16-tenant-resource-quota-warning/design.md)

## 1. 目标

- 到期前 30、7、1 天向启用的租户管理员发送一次站内提醒。
- 到期后自动持久化 `Expired` 状态并使租户全部会话失效。
- 平台管理员可续期、选择是否同时启用，并查看生命周期历史。
- 所有提醒和状态操作可追踪且重复扫描幂等。

## 2. 数据模型与接口

`mini_tenant_lifecycle_records` 保存事件类型、来源、操作人、状态变化、到期时间变化、提醒档位和去重键。提醒去重键由租户、到期时间和提醒档位组成并建立唯一索引。

```http
GET  /platform/tenant/{id}/lifecycle-records
POST /platform/tenant/{id}/renew
```

续期请求：

```json
{
  "expireAt": "2027-07-17T23:59:59+08:00",
  "reactivate": true,
  "remark": "年度续期"
}
```

## 3. 核心决策

| 决策 | 方案 | 理由 |
| --- | --- | --- |
| 提醒幂等 | 生命周期记录唯一去重键 | 同时满足审计和去重，避免冗余状态表 |
| 会话处理 | 复用安全戳、授权缓存和在线会话失效链路 | 已有 token 会立即失效 |
| 权限 | 历史复用 `platform:tenant:query`，续期复用 `platform:tenant:update` | 不增加低收益权限颗粒 |
| 通知 | 租户管理员接收临期提醒，平台管理员接收到期通知 | 过期租户用户无法再进入消息中心 |

## 4. 验收标准

| 验收条件 | 验收方式 |
| --- | --- |
| 同一到期时间的同一提醒档位只发送一次 | 集成测试连续扫描两次 |
| 到期租户状态自动落为 `Expired` | 执行生命周期扫描并查询数据库 |
| 自动过期后旧 token 不可继续访问 | 集成测试调用受保护接口返回 401 |
| 续期可恢复租户并留下记录 | 调用续期接口后查询历史 |
| 前后端和文档可构建 | `dotnet test`、前端 build、VitePress build |

## 5. 暂不实现

| 功能 | 理由 |
| --- | --- |
| 订单、支付、自动扣费 | 超出通用后台框架必要范围 |
| 邮件、短信提醒 | 当前站内通知已满足最小闭环 |
| 自定义提醒天数 | 固定 30/7/1 天足够覆盖通用场景 |
