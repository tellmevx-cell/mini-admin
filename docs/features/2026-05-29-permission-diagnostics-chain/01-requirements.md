# 权限诊断链路增强需求

## 背景

当前系统已经支持 RBAC、租户套餐、菜单权限、数据权限和授权缓存。近期排查租户用户无菜单、登录后 404 时，发现管理员需要同时理解多层规则：

- 用户是否启用
- 用户拥有哪些角色
- 角色是否分配菜单
- 租户套餐是否包含对应菜单
- 最终菜单是否来自角色与套餐的交集
- 授权缓存是否仍是旧数据

现有权限诊断能看到用户、角色、权限码、菜单和缓存 key，但还不能明确说明“卡在哪一层”。

## 目标

- 在现有 `系统监控 / 权限诊断` 页面上增强，不新增菜单。
- 输入用户名后展示完整权限链路：用户、角色贡献、租户套餐上限、最终有效菜单、数据权限、缓存。
- 对租户用户展示套餐名称、套餐菜单数量、最终菜单数量。
- 对每个角色展示角色菜单数量、可见菜单数量、按钮权限数量、是否启用。
- 给出诊断提示，说明常见异常原因：
  - 用户被禁用
  - 没有可用角色
  - 角色没有菜单
  - 套餐没有菜单
  - 角色菜单被租户套餐全部过滤
- 保留刷新缓存能力。

## 非目标

- 不自动给角色分配权限。
- 不绕过租户套餐限制。
- 不改变 RBAC 规则。
- 不新增复杂工作流审批。

## 权限链路

```mermaid
flowchart LR
    A["用户"] --> B["启用角色"]
    B --> C["角色菜单集合"]
    A --> D["租户套餐菜单上限"]
    C --> E["取交集"]
    D --> E
    E --> F["最终菜单与权限码"]
    F --> G["前端菜单/按钮"]
```
