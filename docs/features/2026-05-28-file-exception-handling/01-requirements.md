# 文件异常处理需求文档

## 背景

文件存储一致性检查已经可以发现数据库文件记录和真实存储不一致，并能在定时任务详情中展示缺失文件。但当前仍缺少处置动作，管理员看到异常后只能人工记住问题，不能在系统内标记文件状态。

## 目标

- 给文件记录增加状态。
- 文件一致性检查发现缺失文件时，自动标记为 `Missing`。
- 文件管理列表展示文件状态。
- 下载 `Missing` 或 `Invalid` 文件时返回不可用提示。
- 管理员可以把异常文件标记为 `Invalid`。

## 状态定义

- `Normal`：正常。
- `Missing`：数据库有记录，但实际存储文件不存在。
- `Invalid`：管理员确认无效。

## 功能范围

- 新增文件状态字段。
- 新增标记无效接口。
- 新增标记无效权限 `system:file:mark-invalid`。
- 文件一致性检查任务自动标记缺失文件。
- 前端文件列表展示状态。
- 前端任务执行详情支持标记无效。

## 不做范围

- 不自动删除数据库记录。
- 不自动删除真实存储文件。
- 不做文件重新上传修复。
- 不做批量标记。

## 验收标准

- [x] 新上传文件状态为 `Normal`。
- [x] 文件一致性检查发现缺失文件后，文件状态变为 `Missing`。
- [x] 下载 `Missing` 文件返回 `409 Conflict`。
- [x] 可以调用接口把文件标记为 `Invalid`。
- [x] 下载 `Invalid` 文件返回 `409 Conflict`。
- [x] 文件列表展示文件状态。
- [x] 任务详情可以对文件执行标记无效。
- [x] 后端完整测试通过。
- [x] 前端构建通过。
