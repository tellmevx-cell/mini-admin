# 工作流催办与消息中心联动需求

## 背景

工作流审批中心已经具备发起、待办、已办、抄送、转办、撤回和消息中心通知能力。下一步先继续完善平台层，把工作流动作和消息中心联动做得更闭环，再进入业务模块。

## 本次范围

- 发起人可对自己发起且仍在审批中的流程发送催办。
- 催办应写入流程动作日志，便于详情时间线追踪。
- 催办应进入消息中心，通知当前待办审批人。
- 前端审批中心在“我的申请”列表提供催办入口。
- 催办接口复用工作流权限体系，并由后端二次校验任务归属和状态。

## 暂不包含

- 催办频控。
- 催办模板配置。
- WebSocket 实时推送。
- 邮件、短信、Webhook 等站外通道。
