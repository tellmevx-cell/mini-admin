# 租户套餐菜单授权总结文档

## 本次完成内容

本阶段完成了 SaaS 租户套餐和菜单授权。平台现在可以维护套餐、给套餐分配菜单和按钮权限，并把套餐绑定到租户。租户用户最终权限会被套餐限制，租户角色分配权限时也只能在套餐范围内选择。

## 核心规则

- 平台用户不受套餐限制。
- 租户用户最终权限为 `角色权限 ∩ 套餐权限`。
- 租户管理员打开角色授权树时，只能看到当前租户套餐内的菜单。
- 租户角色保存权限时，后端会过滤套餐外菜单。
- 平台收缩套餐菜单时，会自动删除使用该套餐的租户角色中超出套餐的 `RoleMenu`。
- 默认套餐从空权限调整为租户管理员基础权限，避免新租户创建后无可用菜单。

## 关键实现

- 新增 `TenantPackageAppService`、`ITenantPackageRepository` 和 `EfTenantPackageRepository`。
- 新增 `/platform/tenant-package/*` 套餐管理和授权接口。
- `EfMenuRepository` 在用户菜单和权限码计算时接入套餐上限。
- `EfRoleRepository` 在角色权限读取和保存时接入套餐上限。
- 租户创建、编辑支持 `PackageId`。
- 租户管理页面新增套餐列和套餐选择框。
- 租户套餐页面从占位页升级为真实管理页面，支持新增、编辑、启停和分配权限。

## 验证结果

- 套餐授权专项测试：2 个测试通过。
- 后端完整测试：115 个测试通过。
- 前端构建：通过。

## 建议验收

1. 使用平台 `admin` 登录。
2. 进入 `系统管理 / 租户套餐`，新建一个只包含部分菜单的套餐。
3. 给某个租户绑定该套餐。
4. 使用该租户管理员登录，进入角色管理，确认分配权限树只显示套餐内菜单。
5. 给租户角色分配权限后，用普通租户用户登录，确认只显示套餐内且角色拥有的菜单。
6. 回到平台收缩套餐权限，再确认租户角色超出权限已被自动清理。
