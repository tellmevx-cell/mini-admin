# 租户初始化模板需求

## 背景

当前系统已经支持 SaaS 租户、租户套餐、租户菜单范围、租户管理员和租户数据隔离。现有租户创建流程可以创建租户管理员，但还缺少一个企业级开通闭环：新租户创建后应自动具备基础组织、岗位、普通角色和初始化状态，避免出现“租户能登录，但基础数据不完整”的情况。

## 目标

- 平台管理员新增租户时可以选择初始化模板。
- 第一阶段提供内置模板：`standard` 标准企业模板。
- 新租户创建后自动初始化：
  - 默认部门：总部、研发部、市场部
  - 默认岗位：部门负责人、开发工程师、销售经理
  - 默认角色：租户管理员、普通员工
  - 管理员用户归属总部和部门负责人岗位
  - 普通员工角色分配基础只读菜单权限
- 租户列表展示初始化模板、初始化状态、初始化时间。
- 初始化失败时接口返回错误，避免创建半成品租户。
- 创建过程保持幂等边界：同一个租户创建请求失败后不会重复制造同名租户。

## 非目标

- 本阶段不做模板可视化编辑。
- 本阶段不做行业模板市场。
- 本阶段不自动创建大量业务示例数据。
- 不绕过租户套餐范围，角色菜单仍受套餐限制。

## 数据流

```mermaid
flowchart LR
    A["平台管理员新增租户"] --> B["选择租户套餐"]
    B --> C["选择初始化模板"]
    C --> D["创建租户和管理员"]
    D --> E["初始化部门/岗位/角色"]
    E --> F["分配菜单权限"]
    F --> G["记录初始化状态"]
    G --> H["租户管理员登录使用"]
```

## 验收标准

- 新增租户时默认选择标准企业模板。
- 创建后的租户列表能看到初始化状态为 `Success`。
- 新租户下存在总部、研发部、市场部。
- 新租户下存在部门负责人、开发工程师、销售经理。
- 新租户下存在普通员工角色，且有基础查询类菜单权限。
- 租户管理员登录后仍能看到套餐允许范围内的菜单。
