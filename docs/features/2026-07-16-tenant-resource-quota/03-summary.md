# 租户资源配额闭环实现总结

## 完成内容

- 新增统一租户资源配额服务，实时统计套餐上限、用户数和文件存储字节数。
- 用户单个新增、导入预检和确认导入均接入用户数配额。
- 文件上传接入存储预检、事务内最终校验和失败补偿；删除文件后立即释放额度。
- `ManagedFile` 增加可空 `TenantId`，租户只能访问本租户文件，平台账号保留全局管理能力。
- 平台租户列表展示实时资源用量、进度和超额状态；套餐页面明确 `0` 表示不限额。
- 配额不足统一返回 `409 Conflict`，并携带可直接展示的业务提示。

## 并发与一致性

- 关系型数据库在事务中锁定当前租户行，再读取实时用量并执行写入。
- InMemory 和单实例环境使用租户级信号量串行写入，保证测试与本地开发行为一致。
- 上传文件在落库前进行第二次配额校验，竞争失败时删除已保存的存储对象。
- 套餐降级不删除历史数据，超额租户只能先删除资源或提高套餐上限后再新增。

## 数据库兼容

- 迁移只为 `mini_files` 增加可空 `TenantId` 和 `IX_mini_files_TenantId` 索引。
- 历史文件保持 `TenantId = null`，不会被删除或自动归属到错误租户。
- 新库由初始迁移先创建 `mini_files`，再应用本次增量迁移；旧库也可直接增量升级。
