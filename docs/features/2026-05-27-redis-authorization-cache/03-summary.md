# Redis 授权缓存完工总结

## 完成内容

- 完成 Redis 授权缓存。
- 完成无 Redis 时的内存缓存 fallback。
- 权限码和菜单等高频授权数据可缓存。
- 权限变更后会清理缓存。
- 权限诊断中心可查看缓存 key 并刷新用户缓存。

## 关键实现

- `CacheOptions` 管理缓存配置。
- `DistributedUserAuthorizationCache` 统一读写缓存。
- 缓存 key 以用户维度组织，例如权限码、菜单等。
- 角色、菜单、用户角色变更后主动清理，避免旧权限继续生效。
