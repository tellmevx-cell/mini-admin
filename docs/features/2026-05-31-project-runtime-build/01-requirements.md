# 项目运行管理打包能力需求

## 背景

项目运行管理已经支持登记项目、启动服务、停止服务、查看实时日志。当前用户反馈日志窗口高度不稳定，查看实时输出时体验不够像控制台；同时希望项目面板支持常见技术栈的打包能力，例如 .NET、Vue、React、uniapp。

## 目标

- 日志控制台在当前页面视窗中保持固定高度，日志内容在控制台内部滚动。
- 实时输出开启时，有新日志后自动滚动到最新行。
- 实时输出关闭时，不强制滚动，方便查看历史日志。
- 在服务级别增加“打包”动作，适用于当前选中的 API、前端服务或其他自定义服务。
- 支持为不同项目类型提供默认打包命令。

## 第一版范围

- .NET 默认打包：`dotnet publish -c Release`
- Vue/Vben 默认打包：`pnpm run build:antd`
- React 默认打包：`pnpm run build`
- uniapp 默认打包：`pnpm run build:h5`
- 打包命令写入独立构建日志，不覆盖运行日志。
- 前端提供运行日志/打包日志切换。
- 点击“打包”后自动切换到打包日志，并开启实时输出。

## 非目标

- 第一版不做完整 CI/CD 流水线。
- 第一版不做构建产物发布、远程服务器部署、Docker 镜像推送。
- 第一版不做 uniapp 多端完整矩阵，只提供可配置的 H5 默认命令。

## 权限

沿用 `system:project-runtime:manage` 执行打包动作。
沿用 `system:project-runtime:log` 查看运行日志和打包日志。
