# 项目运行管理服务配置需求

## 背景

项目运行管理已支持服务启动、停止、日志、打包、构建历史和产物管理。当前服务配置主要依赖默认推断和本地 JSON，新增 React、uniapp、其他 .NET 项目时不够直观。

## 目标

- 在页面中新增、编辑、删除工作区服务。
- 支持可视化配置运行命令、端口、健康检查、运行日志。
- 支持可视化配置打包命令、打包日志、构建产物。
- 支持按服务类型套用默认模板：.NET、Vue、React、uniapp、自定义。
- 保存后立即刷新页面配置，不需要手改 JSON。

## 第一版范围

- 复用已有 `PUT /system/project-runtime/projects/{projectId}` 保存整个项目配置。
- 不新增单独服务配置 API。
- 服务配置弹窗覆盖常用字段。
- 删除服务只删除本地配置，不删除文件和产物。
