# Code Generator Workflow Guide Summary

## 完成内容

- 后端 `GenerateAsync` 在进入文件写入和历史落库阶段后，不再使用 HTTP 请求取消令牌，避免前端热更新刷新导致半生成状态。
- 前端生成成功后会保存返回的历史记录，并自动切换到历史页签，按本次模块名过滤记录。
- 如果生成时启用了审批接入，会弹出“审批接入下一步”说明，告诉用户需要先重启后端，再去工作流中心绑定业务类型，然后到业务页面提交审批，最后由审批人在待办中处理。

## 关键说明

生成后表显示“已映射”是正常现象，说明新实体配置已经被系统识别。真正异常的是“文件已生成但没有历史记录”，这次已经针对该问题做了保护。

## 后续方向

可以继续增强为生成成功后给出“一键进入业务绑定”的入口，甚至在生成时选择已有流程并自动创建业务绑定。
