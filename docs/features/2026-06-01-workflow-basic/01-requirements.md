# 工作流一期需求文档

## 背景

MiniAdmin 已具备用户、角色、菜单权限、租户、审计日志、通知和代码生成器等基础能力。下一阶段需要补齐企业后台常见的审批流能力，让系统可以承载请假、报销、采购、合同等需要多人处理的业务。

## 目标

实现一个可用的通用审批工作流一期版本：

- 管理员可以维护流程定义和审批节点。
- 用户可以选择流程并发起审批。
- 审批人可以在我的待办中同意或驳回。
- 发起人可以撤回仍在审批中的实例。
- 用户可以查看流程实例、当前节点、处理状态和流转记录。
- 数据按租户隔离，平台侧和租户侧互不混用。
- 所有新增、审批、驳回、撤回操作走现有审计日志链路。

## 一期范围

### 流程定义

- 字段：名称、编码、表单名称、说明、启用状态。
- 节点：节点名称、审批人类型、指定用户、指定角色、排序。
- 审批人类型一期支持：
  - 指定用户。
  - 指定角色。

### 流程实例

- 字段：标题、流程定义、发起人、业务标识、表单数据、当前节点、状态、开始时间、结束时间。
- 状态：
  - 审批中。
  - 已通过。
  - 已驳回。
  - 已撤回。

### 待办任务

- 为当前节点的审批人生成待办。
- 角色审批节点会给该角色下的所有用户生成待办。
- 任一审批人通过后，当前节点完成，其余同节点待办自动关闭，流程进入下一节点。
- 任一审批人驳回后，流程结束为已驳回。

### 节点式设计器

- 流程定义需要支持节点式、可拖拉画布。
- 前端画布负责节点布局、连线和属性配置。
- 后端使用 .NET 保存画布 JSON，并根据节点连线执行流程流转。
- 一期画布支持：
  - 开始节点。
  - 审批节点。
  - 结束节点。
- 审批节点支持指定用户和指定角色。

### 前端页面

一期做一个聚合页“审批中心”，包含四个标签页：

- 流程定义。
- 发起审批。
- 我的待办。
- 流程实例。

流程定义标签页内提供左侧节点面板、中间画布、右侧节点属性配置。后续再拆成多个菜单页面。

## 非目标

- 不做完整 BPMN 标准设计器。
- 不做条件分支、会签比例、并行网关。
- 不直接绑定代码生成器业务模块。
- 不做移动端审批。

## 验收标准

- admin 可以看到“工作流 / 审批中心”菜单。
- admin 可以新增启用流程定义，并配置用户或角色节点。
- 用户可以发起审批实例。
- 节点审批人可以在待办中审批或驳回。
- 发起人可以撤回审批中的实例。
- 实例详情能看到表单数据和流转记录。
- MySQL 和内存模式均可启动。
