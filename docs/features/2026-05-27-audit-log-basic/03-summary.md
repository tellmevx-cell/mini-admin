# 审计日志基础能力完工总结

## 完成内容

- 完成请求级审计日志采集。
- 审计日志记录用户、接口、方法、IP、耗时、状态码、请求体等信息。
- 完成审计日志列表查询和导出。
- 前端新增审计日志页面。
- 日志删除策略确定为保留 90 天，不提供手动删除。

## 关键实现

- `AuditLogMiddleware` 是请求入口，统一采集审计信息。
- `AuditLogAppService` 负责应用层保存。
- `EfAuditLogRepository` 负责数据库读写。
- 后续实体变更审计通过 `AuditLogId` 与请求日志关联。

## 后续建议

- 对敏感字段做脱敏规则。
- 增加按操作结果、用户、路径、时间范围组合查询。
- 增加后台定时清理 90 天前日志。
