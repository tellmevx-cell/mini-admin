# 安全策略配置需求文档

## 背景

安全中心已经可以查看账号、登录、权限和在线会话风险，但登录失败次数、验证码触发次数、锁定时长、在线活跃窗口等策略仍主要来自 `appsettings`。这不适合企业级后台，因为运维和管理员需要在运行期通过页面调整策略，而不是改配置文件、重启服务。

## 目标

本功能提供一个可视化的安全策略配置入口，让管理员可以在后台维护安全策略。运行时策略优先读取数据库中的系统参数；数据库未配置或配置无效时，使用 `appsettings` 中的默认值兜底。

## 范围

- 在安全中心页面增加“安全策略”配置区。
- 支持读取当前生效策略。
- 支持管理员更新策略。
- 策略保存到系统参数表，分组为 `security`。
- 登录安全服务读取数据库策略并立即影响后续登录校验。
- 安全中心概览中的长期未登录统计、在线会话统计使用策略值。

## 策略项

| 策略 | 参数键 | 默认值 | 说明 |
| --- | --- | --- | --- |
| 验证码触发失败次数 | `security.login.captcha_required_failures` | `3` | 同一用户/IP 连续失败达到该次数后要求验证码 |
| 账号锁定失败次数 | `security.login.lockout_failures` | `5` | 连续失败达到该次数后锁定登录 |
| 账号锁定分钟数 | `security.login.lockout_minutes` | `10` | 锁定持续时间 |
| 验证码有效秒数 | `security.login.captcha_expire_seconds` | `120` | 验证码缓存有效期 |
| 在线活跃分钟数 | `security.online.active_timeout_minutes` | `30` | 超过该时间无请求则不视为在线 |
| 在线心跳写入间隔秒数 | `security.online.touch_throttle_seconds` | `30` | 降低每次请求刷新在线状态的写库频率 |
| 长期未登录天数 | `security.account.stale_user_days` | `90` | 安全中心统计长期未登录用户的阈值 |

## 权限

- 查询策略：`system:security-center:query`
- 更新策略：`system:security-center:update`

当前安全中心已有查询权限，本次增加更新权限并授予管理员角色。

## 数据流

```mermaid
flowchart LR
    A["管理员打开安全中心"] --> B["GET /system/security-policy"]
    B --> C["读取系统参数 security.*"]
    C --> D{"参数存在且有效?"}
    D -- "是" --> E["返回数据库策略"]
    D -- "否" --> F["使用 appsettings 默认值"]
    E --> G["页面展示"]
    F --> G
    G --> H["管理员保存"]
    H --> I["PUT /system/security-policy"]
    I --> J["校验范围"]
    J --> K["写入系统参数"]
    K --> L["后续登录/在线/安全中心统计即时生效"]
```

## 非目标

- 本阶段不做多端会话管理。它会作为下一阶段功能单独设计。
- 本阶段不做复杂审批流，例如安全策略变更二次确认。
- 本阶段不做策略历史版本。操作日志已经能追踪谁修改了接口请求。

## 验收标准

- 管理员可以在安全中心看到当前安全策略。
- 管理员可以修改策略，刷新页面后仍保留。
- 策略修改后，登录验证码触发、账号锁定、在线统计、长期未登录统计读取新值。
- 普通无权限用户不能更新策略。
- 后端测试通过，前端构建通过。
