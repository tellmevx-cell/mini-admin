# 工作流与消息中心验收手册

## 1. 验收目标

这份手册用于确认 MiniAdmin 的工作流和消息中心平台能力已经满足日常使用。

验收范围只包含平台能力：

- 工作流入口、流程定义、流程发布。
- 发起审批、审批通过、审批驳回、撤回、转办、催办。
- 我的申请、我的待办、我的已办、我的抄送。
- 抄送已读/未读追踪。
- 站内信、通知策略、订阅偏好、消息模板、投递记录和失败重试。

验收范围不包含具体业务模块和外部业务平台接入。

## 2. 验收前准备

### 2.1 环境准备

后端：

```powershell
dotnet run --project src/MiniAdmin.Api/MiniAdmin.Api.csproj --urls http://localhost:5021
```

健康检查：

```powershell
Invoke-WebRequest -Uri http://localhost:5021/health -UseBasicParsing
```

前端：

```powershell
cd frontend/vue-vben-admin
pnpm run dev:antd
```

默认地址：

```text
http://localhost:5666
```

默认账号：

```text
admin / 123456
```

### 2.2 基础数据准备

确认以下数据存在：

- 至少一个管理员账号。
- 至少一个可用审批人。
- 至少一个可用被抄送人。
- 至少一个已启用流程定义。
- 通知策略中工作流站内信通道已启用。

如果使用 InMemory 模式，重启后端会重新初始化默认数据。

## 4. 手工验收清单

### A01 菜单和权限入口

操作：

1. 登录管理员账号。
2. 打开 `工作流 / 审批中心`。
3. 打开顶部铃铛。
4. 打开 `系统管理 / 消息通知中心`。

通过标准：

- 审批中心页面可打开。
- 消息中心页面可打开。
- 顶部铃铛可打开消息列表。
- 页面无接口 401、403、404 或前端白屏。

### A02 流程定义可用

### A03 发起审批

操作：

1. 进入 `发起审批`。
2. 选择已启用流程。
3. 填写业务标识，例如 `WF-ACCEPT-001`。
4. 填写审批标题，例如 `平台验收审批`。
5. 填写表单 JSON：

```json
{
  "days": 5,
  "reason": "平台验收测试",
  "type": "测试"
}
```

6. 点击发起审批。

通过标准：

- 提交成功。
- `我的申请` 出现该流程。
- 审批人 `我的待办` 出现任务。
- 审批人收到工作流站内信。

## 6. 上线放行标准

全部满足时可以放行：

- A01 到 A14 全部通过，或失败项已有明确规避方案。
- 后端关键测试通过。
- 前端类型检查通过。
- 前端构建通过。
- 工作区没有意外改动。
- 管理员确认不需要继续增加低收益平台功能。

任一情况不建议放行：

- 发起审批失败。
- 审批通过或驳回无法完成。
- 消息中心无法打开。
- 工作流站内信完全不生成。
- 抄送已读/未读状态不可信。
- 权限导致普通用户能看到不属于自己的待办或抄送。
