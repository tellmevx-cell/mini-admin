# 工作流与消息中心操作手册

## 1. 手册定位

这份手册面向 MiniAdmin 的管理员、审批发起人、审批人和被抄送人，用来说明工作流和消息中心的日常使用方式。

本手册只覆盖平台能力：

- 流程定义、流程发布和流程启停。
- 发起审批、处理待办、查看申请、查看抄送、查看流程详情。
- 消息中心的站内信、模板、通知策略、订阅偏好和投递记录。
- 常见问题排查。

本手册不覆盖具体业务模块接入，也不包含外部业务平台对接。

## 2. 角色说明

| 角色 | 主要操作 |
| --- | --- |
| 系统管理员 | 维护菜单权限、流程定义、通知策略、消息模板和投递记录 |
| 流程管理员 | 创建流程、配置节点、发布流程、维护业务绑定 |
| 发起人 | 发起审批、查看我的申请、撤回待审批流程、催办当前审批人 |
| 审批人 | 查看我的待办、同意、驳回、转办、查看我的已办 |
| 被抄送人 | 查看我的抄送、打开详情后完成已读确认 |

## 3. 登录和入口

1. 打开前端页面。
2. 使用管理员或被授权账号登录。
3. 在左侧菜单进入 `工作流 / 审批中心`。
4. 在顶部铃铛或 `系统管理 / 消息通知中心` 查看消息。

如果看不到入口，优先确认：

- 当前账号是否有对应角色。
- 角色是否分配了工作流和消息中心菜单。
- 退出后重新登录，刷新前端菜单缓存。

## 4. 审批中心页签说明

| 页签 | 用途 |
| --- | --- |
| 我的待办 | 审批人处理待审批任务 |
| 我的申请 | 发起人查看自己提交的流程，可查看详情、撤回或催办 |
| 我的抄送 | 被抄送人查看抄送记录，支持已读和未读追踪 |
| 发起审批 | 选择流程并提交一条新的审批实例 |
| 流程实例 | 管理员查看全部流程运行记录 |
| 流程定义 | 配置流程画布、节点、连线和发布状态 |
| 业务绑定 | 维护通用业务类型和流程定义的绑定关系 |
| 我的已办 | 审批人查看已处理任务 |

## 5. 创建和发布流程

进入 `工作流 / 审批中心 / 流程定义`。

### 5.1 配置基础信息

必填项：

- 流程名称：给用户看的名称，例如 `请假审批示例`。
- 流程编码：系统识别用编码，例如 `leave_apply_demo`。
- 表单名称：说明发起时填写的表单含义。
- 启用状态：启用后才可用于发起审批。

### 5.2 使用画布

画布用于把审批过程可视化。节点代表流程中的一个步骤，连线代表步骤之间的流转方向。

常见节点：

| 节点 | 作用 |
| --- | --- |
| 开始节点 | 流程入口，发起审批后从这里开始 |
| 审批节点 | 生成待办任务，由指定审批人处理 |
| 条件节点 | 根据表单字段或规则选择不同出口 |
| 抄送节点 | 给指定人员生成抄送记录和消息，不阻塞流程继续流转 |
| 结束节点 | 流程终点，表示流程完成 |

常见连线：

| 连线 | 作用 |
| --- | --- |
| 普通连线 | 表示固定的下一步 |
| 条件分支连线 | 表示满足条件时走该出口 |
| 默认分支连线 | 条件都不命中时的兜底出口 |

### 5.3 配置审批节点

审批节点需要确认：

- 节点名称清晰，例如 `直属主管审批`。
- 审批人范围明确，例如指定用户、角色或主管。
- 是否允许转办，按实际管理要求决定。

### 5.4 配置条件分支

条件分支用于把不同情况走向不同路径。例如：

```json
{
  "field": "days",
  "operator": ">",
  "value": 3
}
```

使用建议：

- 条件字段必须来自发起审批时提交的表单 JSON。
- 至少保留一条默认分支，避免没有命中规则时流程无法继续。
- 条件名称要写清楚，例如 `请假大于 3 天`、`默认分支`。

### 5.5 配置抄送节点

抄送节点用于通知相关人员知晓流程进展。

使用建议：

- 只抄送确实需要知晓的人，避免消息噪音。
- 抄送不等于审批，被抄送人不需要同意或驳回。
- 被抄送人打开详情后，系统会记录已读时间。

### 5.6 保存、整理连线和发布

1. 点击 `保存草稿`。
2. 使用 `整理连线` 检查画布是否清晰。
3. 确认每个节点都有正确入口和出口。
4. 发布流程。
5. 确认流程处于启用状态。

## 6. 发起审批

进入 `工作流 / 审批中心 / 发起审批`。

操作步骤：

1. 选择已启用流程。
2. 填写业务标识，用于后续检索，例如 `LEAVE-20260609-001`。
3. 填写审批标题。
4. 填写表单数据 JSON。
5. 点击发起审批。

示例表单：

```json
{
  "days": 5,
  "reason": "家中有事，需要请假",
  "type": "事假"
}
```

提交成功后：

- 发起人可以在 `我的申请` 看到记录。
- 审批人可以在 `我的待办` 看到任务。
- 相关人员会收到站内信。

## 7. 处理待办

进入 `工作流 / 审批中心 / 我的待办`。

常用操作：

| 操作 | 效果 |
| --- | --- |
| 详情 | 打开流程详情，查看表单、当前任务、审批记录和抄送回执 |
| 同意 | 当前审批节点通过，流程进入下一步或结束 |
| 驳回 | 流程被驳回，发起人收到驳回通知 |
| 转办 | 将当前待办转给其他审批人处理 |

处理建议：

- 同意或驳回时填写清楚意见。
- 转办前确认接收人有处理职责。
- 如果详情中流程数据异常，不要继续审批，先联系管理员排查流程定义。

## 8. 查看我的申请

进入 `工作流 / 审批中心 / 我的申请`。

可以查看：

- 我发起的流程。
- 当前状态。
- 当前节点。
- 完成时间。
- 流程详情和流转记录。

可用操作：

- 对仍在审批中的流程进行撤回。
- 对仍在审批中的流程进行催办。
- 查看已完成、已驳回或已撤回流程的完整记录。

## 9. 查看我的抄送

进入 `工作流 / 审批中心 / 我的抄送`。

使用方式：

1. 默认查看全部抄送。
2. 使用已读/未读筛选定位待阅知记录。
3. 打开详情后，系统自动标记为已读。
4. 回到列表后，未读数量和状态会刷新。

注意事项：

- 抄送只代表知会，不需要审批动作。
- 已读时间会在流程详情的抄送回执中展示。
- 如果从消息中心点击抄送消息进入详情，也会触发已读。

## 10. 使用消息中心

进入 `系统管理 / 消息通知中心`。

### 10.1 我的消息

用于查看当前用户收到的站内信。

常用操作：

- 按已读/未读筛选。
- 打开消息详情。
- 点击工作流消息跳转到对应流程详情。
- 将消息标记为已读。

### 10.2 通知策略

用于控制不同事件是否发送通知，以及发送到哪些通道。

建议：

- 工作流核心事件默认保留站内信。
- 邮件和 Webhook 只有在配置完整后再启用。
- 关闭全局策略后，个人订阅不能强行打开该通道。

### 10.3 订阅偏好

用于让个人调整自己想接收的事件。

建议：

- 普通用户只调整确实影响自己的通知。
- 出现收不到消息时，可先点击恢复默认。

### 10.4 模板配置

用于维护通知标题、内容和跳转链接。

建议：

- 修改模板后，发一条测试流程验证渲染结果。
- 保留必要变量，不要删除流程跳转需要的链接参数。
- 模板文案应简洁，能看出流程标题、状态和处理人。

### 10.5 投递记录

用于排查邮件、Webhook 等外部通道是否发送成功。

常见状态：

| 状态 | 含义 |
| --- | --- |
| Sent | 已发送 |
| Failed | 发送失败 |
| Skipped | 因配置或策略跳过 |
| Pending | 等待发送或等待重试 |

处理建议：

- 站内信正常但邮件失败时，优先检查邮件配置。
- Webhook 失败时，优先检查地址、网络和鉴权。
- 修复配置后，再对失败记录执行重试。

## 11. 管理员日常检查

建议每次上线前检查：

- 工作流菜单和消息中心菜单可见。
- 管理员账号能看到流程定义、流程实例、通知策略、模板配置和投递记录。
- 普通审批人只能看到自己相关的待办、已办、申请和抄送。
- 至少存在一个已启用流程用于冒烟测试。
- 通知策略中站内信通道保持启用。
- 邮件和 Webhook 没有配置完整时不要误认为外部发送成功。

## 12. 常见问题

### 12.1 看不到菜单

处理顺序：

1. 确认角色已分配菜单。
2. 退出重新登录。
3. 检查初始化种子数据是否执行。
4. 检查租户或套餐是否限制了菜单。

### 12.2 发起审批后没有待办

处理顺序：

1. 确认流程已发布并启用。
2. 确认开始节点连到了审批节点。
3. 确认审批节点能解析出审批人。
4. 检查流程详情中的当前节点和流转日志。

### 12.3 条件分支没有按预期走

处理顺序：

1. 检查表单 JSON 字段名是否和条件字段一致。
2. 检查数字和字符串类型是否写对。
3. 检查是否存在默认分支。
4. 检查多个条件的优先级和出口连线。

### 12.4 消息没有生成

处理顺序：

1. 检查通知策略是否启用该事件。
2. 检查个人订阅偏好是否关闭。
3. 检查工作流动作是否实际发生。
4. 检查消息是否被幂等规则去重。

### 12.5 抄送没有变成已读

处理顺序：

1. 确认打开的是抄送详情，而不是普通流程实例详情。
2. 确认当前用户就是被抄送人。
3. 从 `我的抄送` 或携带抄送参数的消息入口重新打开。
4. 刷新列表后再检查状态。

## 13. 收口原则

当前平台能力已经覆盖基础使用闭环。除非实际使用中出现明确痛点，否则不建议继续扩展低频功能。

暂不纳入本阶段：

- 加签和会签。
- 委托代理审批。
- 复杂 SLA 超时规则。
- 统计大屏。
- 任意表达式规则引擎。
- 具体业务模块功能。
